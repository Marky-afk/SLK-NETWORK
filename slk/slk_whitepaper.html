1. Abstract

SLK is a peer-to-peer decentralized financial system built on a novel consensus mechanism called Proof-of-Race (PoR). Unlike Proof-of-Work which wastes energy on arbitrary computation, SLK mining is tied to physics-based race simulations using Poisson processes, Lorenz attractors, and SHA3-256 verification — making every mined block the result of real deterministic effort.

SLK combines a fixed-supply blockchain with a complete financial ecosystem: commercial banks, reserve banks, loans, escrow, multi-signature wallets, time-locked transactions, recurring payments, smart contracts, and on-chain governance — all running on a real libp2p P2P network with Kademlia DHT discovery.

"SLK treats money as math. Every coin is traceable to its origin block. Every transaction is cryptographically signed. No inflation. No central authority. No exceptions."

2. The Problem

The global financial system excludes billions of people. Traditional banking requires documentation, credit history, and physical presence. Bitcoin solved digital scarcity but created a mining arms race that wastes electricity. Ethereum introduced smart contracts but gas fees make small transactions impractical.

SLK solves this differently:

  • Mining requires computation but not wasteful hashing — Proof-of-Race has a natural difficulty ceiling
  • On-chain banking means anyone with a phone can create or use a bank
  • Fixed supply of 2 billion SLK ensures no inflation ever dilutes holdings
  • Smart contracts execute automatically with zero gas fees between peers

3. Proof-of-Race Consensus

Each block in the SLK chain is called a Trophy. Trophies are earned by winning a physics-based race simulation. The race uses:

  • Poisson processes — random events model real-world race variance
  • Lorenz attractor dynamics — chaotic but deterministic physics engine
  • VDF (Verifiable Delay Function) — proves elapsed real time, preventing instant fake wins
  • SHA3-256 verification — each trophy hash is verifiable by any node

Reward Tiers

  • 🥇 Gold — finish within target time: full 0.008 SLK reward
  • 🥈 Silver — finish within 130% of target: 0.0056 SLK
  • 🥉 Bronze — any finish: 0.0032 SLK

Block Hashing

SLK uses double SHA-256 on all block hashes — identical to Bitcoin — making length-extension attacks mathematically impossible. Each block includes the previous block hash, winner address, race distance, finish time, and VDF proof.

4. P2P Network Architecture

SLK runs on libp2p with Kademlia DHT peer discovery. Every node is equal — there are no master nodes or validators. The network uses GossipSub for message propagation with the following topics:

  • slk-trophies — new blocks broadcast instantly to all peers
  • slk-transactions — mempool transactions
  • slk-bank-records — bank deposits, loans, transfers
  • slk-social-feed — decentralized social layer
  • slk-exchange-orders — P2P exchange order book

All messages are capped at 64KB. Rate limiting prevents spam. New nodes sync the full chain from peers on startup via the /slk/chainsync/1.0.0 protocol.

5. UTXO Model & Transactions

SLK uses the Unspent Transaction Output (UTXO) model — every coin is traceable to the trophy block that created it. Double-spend protection is enforced at the UTXO level: once an output is spent, it cannot be spent again.

Transaction signing uses Ed25519 — 128-bit security level. Breaking a single key would require more computation than exists in the observable universe. All transactions include sender, recipient, amount, fee, timestamp, Ed25519 signature, and public key.

The mempool enforces: positive amounts, valid addresses, no self-sends, replay attack protection (10-minute window), per-sender rate limiting (100 pending max), and total size cap (10,000 transactions).

6. On-Chain Banking System

Any SLK holder can create a bank. Two bank types exist:

Commercial Banks

Issue a custom currency backed 1:1 by locked SLK. Set exchange rates, fee basis points, and interest rates. Accept deposits from other network participants. Earn fees on every transaction. Broadcast their existence to all peers.

Reserve Banks

Lock SLK as reserves and issue currency up to a reserve ratio. Function like real central banks — control monetary supply, set lending rates, earn seigniorage. Reserve banks can lend to commercial banks.

Banking Features

  • Deposits with compound interest
  • Peer-to-peer loans with configurable terms
  • B2B transfers between banks
  • Currency exchange at market rates
  • Website API for external integration

7. Smart Contracts

SLK smart contracts are programmable conditions on transactions. Six contract types are supported:

  • Escrow — funds released when both parties confirm
  • Vesting — funds released linearly over time (employee salaries, investor lockups)
  • MultiSig — N-of-M signatures required (company treasuries)
  • Conditional — funds released when creator marks condition met
  • Savings — lock SLK until savings target reached
  • Will — release to beneficiary after inactivity period

Contracts execute automatically every 30 seconds. Locked SLK is deducted immediately on creation. Cancelled contracts return funds to creator. All executions are recorded as transactions and broadcast to the network.

9. Security Model

  • Ed25519 signatures — 128-bit security, quantum-resistant for foreseeable future
  • Double SHA-256 blocks — length-extension attacks impossible
  • Merkle trees — transaction inclusion provable without full chain download
  • Atomic disk writes — power failure cannot corrupt chain state
  • Replay protection — 10-minute transaction window prevents replay attacks
  • P2P rate limiting — 64KB message cap, 100 tx/sender mempool limit
  • UTXO double-spend protection — every output tracked, spent outputs rejected
  • Integer arithmetic — all amounts stored as nanoSLK (int64) preventing float rounding

10. Governance

Protocol changes require an on-chain governance vote. Any holder with 10+ SLK can submit a proposal. Voting weight equals SLK balance. Proposals pass by simple majority within the voting window. A time-lock delay between passing and activation gives the network time to upgrade.

11. Decentralized Identity

Businesses, NGOs, and governments can register a verified identity on-chain. Registration is voluntary and stores organisation name, type, registration number, and website. Verified wallets display a badge visible to all network participants, enabling trust without a central authority.
